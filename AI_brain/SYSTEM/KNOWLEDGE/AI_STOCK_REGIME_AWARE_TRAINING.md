# AI_STOCK_REGIME_AWARE_TRAINING.md

## Purpose

This document defines the **Regime‑Aware Training methodology** for the
CatBoost model used in the AI_STOCK system.

The objective is to ensure the model performs consistently across
different market conditions:

-   Bull market
-   Bear market
-   Sideways market

Financial markets are **regime dependent systems**.\
Patterns that work in bull markets often fail in bear markets.

Regime‑aware training ensures the model understands **market context**
before making predictions.

Applicable model: **R7 CatBoost**

------------------------------------------------------------------------

# 1. Core Concept

Standard ML models assume the data distribution is stable.

Financial markets violate this assumption.

Market conditions shift between:

  Regime        Description
  ------------- -----------------------------------
  Strong Bear   Panic selling, liquidity collapse
  Bear          Downtrend with lower highs
  Neutral       Sideways / consolidation
  Bull          Uptrend with pullbacks
  Strong Bull   Momentum driven rally

The model must learn **different rules for different regimes**.

------------------------------------------------------------------------

# 2. Regime State Feature

Introduce a feature:

    regime_state

Range:

    -4 → +4

Example:

  Market State   Value
  -------------- -------
  Strong Bear    -4
  Bear           -2
  Neutral        0
  Bull           +2
  Strong Bull    +4

This feature is generated by the **R4_REGIME module**.

The regime feature is included in the **Meta Layer Feature Matrix**.

------------------------------------------------------------------------

# 3. Regime Feature Interaction

CatBoost can automatically learn interactions between regime and
technical features.

Example learned rule:

    RSI < 35 AND regime_state > 1 → bullish reversal

But:

    RSI < 35 AND regime_state < -2 → bearish continuation

This allows the model to **interpret the same indicator differently
depending on market context**.

------------------------------------------------------------------------

# 4. Sample Weighting

Bear markets occur less frequently than bull markets.

To avoid model bias, apply **sample weighting** during training.

Example weighting:

  Regime     Weight
  ---------- --------
  Bull       1
  Sideways   1.5
  Bear       3

Implementation example:

``` python
sample_weight = regime_weights
model.fit(X, y, sample_weight=sample_weight)
```

This ensures the model does not ignore bear market behavior.

------------------------------------------------------------------------

# 5. Additional Regime Features

Additional contextual features can improve regime awareness.

### Regime Duration

    regime_duration

Number of days the current regime has persisted.

Long regimes increase probability of **mean reversion**.

------------------------------------------------------------------------

### Regime Transition

    regime_transition

Measures regime change velocity.

Example:

  Transition    Value
  ------------- -------
  Bull → Bear   -3
  Bear → Bull   +3
  Stable        0

Market volatility often spikes during regime transitions.

------------------------------------------------------------------------

### Regime Volatility Context

    volatility_regime_mode

Indicates volatility state:

-   Low volatility
-   Normal volatility
-   High volatility

Volatility regimes significantly influence trading outcomes.

# 7. CatBoost Training Configuration

Example CatBoost configuration:

``` python
model = CatBoostClassifier(
    iterations=2000,
    depth=6,
    learning_rate=0.02,
    l2_leaf_reg=8,
    loss_function="Logloss",
    eval_metric="AUC"
)
```

Training pipeline:

    Load Feature Matrix
    ↓
    Add regime_state feature
    ↓
    Apply sample weighting
    ↓
    Train CatBoost model
    ↓
    Evaluate by regime

------------------------------------------------------------------------

# 8. Evaluation by Regime

Model performance must be evaluated separately for each regime.

Example metrics:

  Regime     Accuracy
  ---------- ----------
  Bull       62%
  Sideways   55%
  Bear       60%

If bear accuracy is too low, adjust:

-   regime weighting
-   regime features
-   feature engineering

------------------------------------------------------------------------

# 9. Benefits

Regime-aware training provides:

-   Higher stability across market cycles
-   Reduced overfitting to bull markets
-   Better risk awareness in bear markets
-   Improved feature interaction learning
